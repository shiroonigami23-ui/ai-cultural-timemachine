# Contributing to AI Cultural Time Machine

We welcome contributions to the AI Cultural Time Machine! By contributing, you help us build a richer, more accurate, and immersive historical experience.

## How to Contribute

### 1. Code Contributions

**Fork the Repository**: Start by forking the `ai-cultural-timemachine` repository to your GitHub account.

**Clone Your Fork**: Clone your forked repository to your local machine:

```bash
git clone https://github.com/YOUR_USERNAME/ai-cultural-timemachine.git
cd ai-cultural-timemachine
```

**Create a New Branch**: For each new feature or bug fix, create a new branch. Use a descriptive name:

```bash
git checkout -b feature/your-new-feature
# or
git checkout -b bugfix/fix-that-bug
```

**Make Your Changes**: Implement your feature, fix the bug, or add new content. Please ensure your code adheres to the existing style and conventions.

**Test Your Changes**: Before submitting, make sure your changes work as expected and don't break existing functionality.

**Commit Your Changes**: Write clear and concise commit messages. A good commit message explains *what* was changed and *why*.

```bash
git commit -m "feat: Add new era selection component"
# or
git commit -m "fix: Resolve loading issue on ModelStatus"
```

**Push to Your Fork**: Push your new branch to your forked repository on GitHub:

```bash
git push origin feature/your-new-feature
```

**Open a Pull Request (PR)**: Go to the original `ai-cultural-timemachine` repository on GitHub and open a new Pull Request from your branch. Please provide:

*   A clear title describing the PR's purpose.
*   A detailed description of the changes you've made.
*   Any relevant screenshots or GIFs for UI changes.
*   References to any issues the PR addresses.

### 2. Historical Data Contributions

We rely heavily on historically accurate data. If you're a historian or enthusiast, you can contribute by:

*   **Providing Text Corpora**: Curated texts, diaries, letters, or academic papers relevant to specific eras.
*   **Suggesting Image References**: Links to public domain art, archaeological findings, or historical photographs.
*   **Fact-Checking**: Reviewing existing AI-generated content for historical accuracy.

Please submit data contributions via GitHub Issues or by opening a PR with your data in the `data/` directory, along with a `README.md` explaining the source and context.

### 3. Reporting Bugs and Suggesting Features

**Report Bugs**: If you find a bug, please open an [issue on GitHub](https://github.com/shiroonigami23-ui/ai-cultural-timemachine/issues). Provide a clear description, steps to reproduce, and expected behavior.

**Suggest Features**: Have an idea for a new era, AI model, or platform improvement? Open an [issue on GitHub](https://github.com/shiroonigami23-ui/ai-cultural-timemachine/issues) to discuss your idea. We love creative suggestions!
